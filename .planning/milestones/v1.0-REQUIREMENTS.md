# Requirements Archive: v1.0 MVP

**Milestone:** v1.0 MVP  
**Shipped:** 2026-05-20  
**Core Value:** Each user's notes are encrypted at rest and invisible to every other user

---

## v1 Requirements — All Complete

### Authentication

- [x] **AUTH-01**: User can sign in with email and password — Phase 1 ✓
- [x] **AUTH-02**: User session persists across browser refresh — Phase 1 ✓
- [x] **AUTH-03**: User can sign out from any page — Phase 1 ✓
- [x] **AUTH-04**: Webmaster can create a user via Rails console (no self-registration UI) — Phase 1 ✓

### Notes Management

- [x] **NOTE-01**: User can create a note with a markdown body — Phase 2 ✓
- [x] **NOTE-02**: User can attach a single file to a note (any file type) — Phase 2 ✓
- [x] **NOTE-03**: User can create a note with only a file attachment (no body required) — Phase 2 ✓
- [x] **NOTE-10**: A note must have at least a body or a file attachment — model validation rejects notes with neither — Phase 2 ✓
- [x] **NOTE-04**: User can edit a note's body and/or replace the file attachment — Phase 2 ✓
- [x] **NOTE-05**: User can delete a note permanently — Phase 2 ✓
- [x] **NOTE-06**: Note title is derived from the first line of the body when body is present — Phase 2 ✓
- [x] **NOTE-07**: Note title falls back to the attachment filename when body is absent — Phase 2 ✓
- [x] **NOTE-08**: User can view a note with the markdown body rendered as HTML — Phase 2 ✓
- [x] **NOTE-09**: User can download a note's file attachment — Phase 2 ✓

### Notes List

- [x] **LIST-01**: User sees a table of their own notes on the main page — Phase 2 ✓
- [x] **LIST-02**: Table shows: derived title, created date, updated date, attachment indicator — Phase 2 ✓
- [x] **LIST-03**: User sees only their own notes (no cross-user visibility) — Phase 2 ✓

### Encryption & Security

- [x] **SEC-01**: Note body is encrypted at rest using Active Record Encryption — Phase 2 ✓
- [x] **SEC-02**: Unauthenticated requests are redirected to the sign-in page — Phase 1 ✓

## Out of Scope — Validated

| Feature | Reason |
|---------|--------|
| Self-registration | Deliberate security decision — webmaster-only user creation |
| Shared / collaborative notes | Strictly private per-user by design |
| Multiple file attachments per note | One attachment only for v1 |
| Search / filter on notes list | Deferred to v2 |
| OAuth / SSO login | Email/password sufficient for controlled deployment |
| Mobile app | Web-first |
