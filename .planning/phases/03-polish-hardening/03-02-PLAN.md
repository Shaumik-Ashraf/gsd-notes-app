<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Rails app | All HTTP responses cross this boundary; baseline headers + CSP enforce client-side defense |
| Web server → public/*.html | Static error pages served when Rails is unavailable — must not depend on app assets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering / XSS | Inline scripts injected into rendered HTML | mitigate | Strict CSP: `script-src :self` + no `unsafe-inline` blocks any injected inline `<script>`. Turbo/Stimulus served via importmap from `:self`. |
| T-03-07 | Spoofing | Clickjacking via iframe embedding | mitigate | `X-Frame-Options: SAMEORIGIN` set globally in security_headers.rb; CSP `frame-ancestors :self` reinforces. |
| T-03-08 | Tampering | MIME sniffing forcing JS execution from non-JS responses | mitigate | `X-Content-Type-Options: nosniff` set globally. |
| T-03-09 | Information disclosure | Referrer leakage to third parties | mitigate | `Referrer-Policy: strict-origin-when-cross-origin` set globally. |
| T-03-10 | Information disclosure | Error pages leaking stack traces / app internals | mitigate | Restyled public/*.html contain only generic copy + app name; no internal paths, links, or telemetry. |
| T-03-11 | DoS | CSP report-uri abuse | accept | No report-uri configured — keeps blast radius small at the cost of telemetry. |
</threat_model>
