# Phase 3 — UI Design Contract

> Visual and interaction contract for the Polish & Hardening phase.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** This phase replaces the blue accent palette (`#2563eb`) established in
> Phase 2 with a strict black-and-white palette per D-UI-01. All pages are restyled;
> no new functional UI is added. The contract below is the **target state** after polish.

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap, label-to-input gap, inline code padding |
| sm | 8px | Table cell padding (vertical), flash message vertical padding, stacked field gap |
| md | 16px | Table cell padding (horizontal), field margin-bottom, button horizontal padding, section gaps |
| lg | 24px | Card/form padding (vertical), page header margin-bottom, action bar gap |
| xl | 32px | Nav horizontal padding, main content horizontal padding |
| 2xl | 48px | Empty state vertical padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets for icon-only edit/delete buttons in the table actions column: minimum 44px × 44px hit area (padding around the icon).
- Nav bar fixed height: 48px (already set, keep).

## Typography

Declared type scale — exactly 4 entries:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 | 1.5 | Table data cells, note body prose, form inputs, textarea |
| Label / Meta | 14px | 400 | 1.5 | Table headers, field labels, meta dates (created/updated timestamps), attachment indicator, nav sign-out, flash text |
| Heading | 20px | 600 | 1.2 | Page `<h1>` (e.g. "Notes"), empty-state heading, note-body h1/h2 |
| Display | 24px | 600 | 1.2 | Sign-in card title only (already set, keep) |

**Weights in use: 400 (body/label) and 600 (heading/display). No other weights.**

Monospace exception: `ui-monospace, SFMono-Regular, "Courier New", monospace` at 14px / weight 400 for inline code and code blocks inside rendered markdown.

**Standalone-file exception (outside the declared type scale):** Public error pages (`public/404.html`, `public/422.html`, `public/500.html`) embed their own `<style>` block and use a 48px error code number. This value is not a scale entry — it applies only inside those self-contained files and must not be referenced anywhere else in the app.

## Color

**Target palette: black-and-white only. All blue (`#2563eb`) references must be replaced.**

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | Page background, card/form backgrounds, table row background |
| Secondary (30%) | `#f5f5f5` | App background (body), table header row, table row hover, code block background |
| Accent (10%) | `#111827` | Primary action buttons (black fill + white text), nav wordmark, body text, headings |
| Destructive | `#dc2626` | Inline validation error text, `.btn-destructive` delete link, `.sign-in-error`, error summary text |

Accent (`#111827`) is reserved for:
1. `.btn-primary` — black fill background, white text
2. `.app-nav-title` — "Notes" wordmark text
3. Body copy (`color: #111827`) — all readable text
4. Table data cell text

**Accent is NOT used for:** links, focus rings as color fill, or decorative borders.

**Muted text:** `#6b7280` for secondary labels, table header text, meta info, placeholder text, nav sign-out link — this is not an accent, it is a supporting neutral.

**Borders:** `#d1d5db` (standard), `#f3f4f6` (table row divider).

**Focus ring:** `outline: 2px solid #111827; outline-offset: 2px;` — replace all `#2563eb` focus ring references.

**Link style:** Replace `color: #2563eb` links with `color: #111827; text-decoration: underline;` for inline links. Nav and action links use `color: #111827` with no underline by default, underline on hover.

**Flash colors — keep existing semantic values:**
- `.flash-notice` — `background: #f0fdf4; color: #166534;` (green, kept as-is — semantic signal, not decorative accent)
- `.flash-alert` — `background: #fef2f2; color: #991b1b;` (red, kept as-is)

## Component Inventory

### Nav Bar (`.app-nav`)
- Height: 48px, `background: #ffffff`, `border-bottom: 1px solid #d1d5db`
- Left: "Notes" wordmark — 16px / weight 600 / `#111827` — plain text, not a link
- Right: current user email at 14px / `#6b7280`, then sign-out button at 14px / `#6b7280` (hover: `#111827`)
- Source: D-UI-05

### Primary Button (`.btn-primary`)
- `background: #111827`, `color: #ffffff`, `border: none`
- Height: 36px, horizontal padding: 16px, border-radius: 6px, font-size: 14px
- Hover: `background: #1f2937`
- Active: `background: #374151`
- Replaces all blue `.btn-primary` instances
- Source: D-UI-06

### Secondary / Ghost Button (`.btn-secondary`)
- `background: #ffffff`, `color: #111827`, `border: 1px solid #d1d5db`
- Same height/padding/radius as `.btn-primary`
- Hover: `border-color: #111827`
- Used for: Cancel, Edit (when alongside a primary action), table action links
- Source: D-UI-04, Claude's Discretion

### Notes Index Table (`.notes-table`)
- Columns: Title | Created | Updated | Attachment | Actions
- Row click → note show page (entire `<tr>` or `<td>:first-child` link wrapping the title)
- Actions column: Edit (pencil icon or "Edit" text link) + Delete (trash icon or "Delete" text link), last column
- Exact icons: Unicode `✎` (U+270E) for edit, `🗑` or text "Delete" for delete — Claude's Discretion to choose inline SVG vs text
- Actions column width: 100px; icon buttons: 44×44px hit area via padding
- Title truncation: `max-width: 420px; overflow: hidden; text-overflow: ellipsis; white-space: nowrap;` (~60–80 chars at 16px)
- Source: D-UI-02, LIST-02

### Note Form (`_form.html.erb`)
- `<textarea>` label: "Note" or "Body", placeholder: "Write your note in markdown"
- Plain styled textarea: no JS editor widget
- Error summary (`#error_explanation`) above fields: 14px / `#dc2626`
- Per-field inline error: `.field-with-errors input` border changes to `#dc2626`
- Source: D-UI-03, D-ERR-01

### Note Show Page Action Bar
- Position: top-right of content area (`.note-header` with flex + space-between)
- Left: note title derived from first markdown line (h1-level heading in the page, 20px/600)
- Right: Edit button (`.btn-secondary`) + Delete button (`.btn-destructive` or ghost with red text)
- Delete uses `data: { turbo_confirm: "Delete this note? This action cannot be undone." }`
- Source: D-UI-04

### Public Error Pages (`public/404.html`, `public/422.html`, `public/500.html`)
- Standalone HTML — no Rails layout, no external stylesheet link
- Embed `<style>` block directly in `<head>`
- Layout: centered vertically and horizontally, `min-height: 100vh`, `background: #f5f5f5`
- App name "Notes" as display heading (24px / weight 600 / `#111827`)
- Error code as large muted number (48px / weight 600 / `#d1d5db`) above the heading — standalone-file exception, not a type scale entry
- Short human message below (16px / 400 / `#6b7280`)
- No navigation links (Rails may be down)
- Source: D-SEC-02

### Validation Error Display
- Summary block at top of form: `<div id="error_explanation">` — 14px, `#dc2626`, no background fill
- Per-field: Rails wraps in `.field_with_errors`; add CSS: `.field_with_errors input, .field_with_errors textarea { border-color: #dc2626; }`
- Source: D-ERR-01
