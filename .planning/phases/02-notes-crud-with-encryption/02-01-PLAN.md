<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Database disk -> filesystem snapshot | The notes table sits on disk. Anyone with read access to `storage/development.sqlite3` (backups, leaked image, host compromise) sees the raw `body` column. Encryption at rest defends this boundary. |
| Developer machine -> credentials.yml.enc | The master.key file decrypts the credentials. master.key is git-ignored; loss = unable to read notes, leak = full decryption capability. |
| User -> Note association | A row's `user_id` is the only thing tying a note to its owner. If scoping is broken in later plans, cross-user disclosure happens. (Enforcement lives in NotesController in Plan 02 — see threat T-02-01 there.) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Information Disclosure | `notes.body` column at rest | mitigate | `encrypts :body` (Active Record Encryption) per D-03/SEC-01; tested via raw-SQL assertion that ciphertext != plaintext in Task 3 acceptance criteria. |
| T-02-01-02 | Tampering | `db/migrate/*_create_notes.rb` schema | mitigate | `body` column is `text` and nullable so NOTE-10 model-level validation is the source of truth; `t.references :user, null: false, foreign_key: true` enforces FK integrity at DB layer. |
| T-02-01-03 | Repudiation | Note ownership | mitigate | `belongs_to :user` is required (Rails 8 default) so a note cannot be orphaned. |
| T-02-01-04 | Denial of Service | Attachment purge on user delete | accept | `dependent: :destroy` on User + `dependent: :purge` on attachment is synchronous; acceptable for v1 scale (D-07). Heavy multi-attachment cases are out of scope. |
| T-02-01-05 | Elevation of Privilege | Credentials leak | mitigate | Keys stored in `credentials.yml.enc`, decrypted by `master.key` which is in `.gitignore`. Documented in CLAUDE.md so future contributors do not commit master.key. |
| T-02-01-SC | Tampering | npm/pip/cargo installs | accept | Only RubyGems change: `redcarpet` (well-known gem, >700k downloads/day). No npm/pip/cargo installs this plan; slopcheck not applicable to Bundler. |
</threat_model>
