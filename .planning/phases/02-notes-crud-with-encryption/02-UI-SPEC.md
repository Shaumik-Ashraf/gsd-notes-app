# Phase 2 — UI Design Contract

> Visual and interaction contract for the notes CRUD surfaces. Generated by gsd-ui-researcher.

---

## Scope

This contract covers four UI surfaces delivered by Phase 2:

1. **Notes index** (`/notes`) — table listing authenticated user's notes
2. **Note show** (`/notes/:id`) — rendered markdown body + attachment download link
3. **Note new/create** (`/notes/new`) — form with textarea and file input
4. **Note edit/update** (`/notes/:id/edit`) — same form, pre-populated

The sign-in page is governed by the Phase 1 UI-SPEC (approved 2026-05-18). All tokens here extend the Phase 1 contract without overriding it.

## Layout

The notes pages use `app/views/layouts/application.html.erb`. Phase 2 adds a minimal page layout wrapper — a top navigation bar and a content area — consistent with an authenticated context.

**Navigation bar** (rendered on all notes pages):
- Height: 48px
- Background: #ffffff
- Bottom border: 1px solid #d1d5db
- Contains: "Notes" app title (left-aligned, 16px weight 600) and "Sign out" link (right-aligned, 14px weight 400, color #6b7280)
- Padding: 0 32px (xl token)
- Sticky top: `position: sticky; top: 0; z-index: 10`

**Page content area**:
- Max-width: 960px
- Margin: 0 auto
- Padding: 24px 32px (lg/xl tokens)

### Notes Index Page

- Page heading "Notes" — 20px weight 600, margin-bottom: 16px
- **Primary focal point:** the "New note" accent button in the heading row (top-right, accent background #2563eb) — this is the single most prominent interactive element on the page
- Notes table — full width within content area
- Empty state — centered in table area when no notes exist

### Note Show Page

- Back link "← Notes" — 14px weight 400 color #6b7280, margin-bottom: 16px
- Note derived title — 20px weight 600, margin-bottom: 8px
- Metadata row — 12px weight 400 color #6b7280; format: "Created {date} · Updated {date}" — below title, margin-bottom: 16px
- Attachment row — 14px weight 400; shown only if attachment exists; "Attachment: {filename}" with a download link styled in accent color (#2563eb); margin-bottom: 16px
- Divider — 1px solid #d1d5db, margin: 16px 0
- Rendered markdown body — standard prose styling (see Typography section); shown only if body present

### Note New / Edit Form Page

- Back link "← Notes" — 14px weight 400 color #6b7280, margin-bottom: 16px
- Page heading — "New note" or "Edit note" — 20px weight 600, margin-bottom: 24px
- Form fields stacked vertically, 16px gap between fields
- Submit button — full-width on mobile, auto-width (min 120px) on desktop; accent background
- Cancel link — next to submit, "Cancel" plain text link in #6b7280, 14px

- `blockquote`: left border 3px solid #d1d5db, padding-left 16px, color #6b7280

## Component Inventory

### Notes Table

- Element: `<table>` with class `notes-table`
- Columns (in order): Title | Created | Updated | File
- Header row: 14px weight 400 color #6b7280, background #f9fafb, bottom border 1px solid #d1d5db
- Data rows: 16px weight 400 color #111827, row bottom border 1px solid #f3f4f6 (lighter than header)
- Row hover: background #f9fafb
- Title cell: clickable link to note show page, color #2563eb, no underline, underline on hover
- Created/Updated cells: 14px color #6b7280; date format "MMM D, YYYY" (e.g. "May 18, 2025")
- File cell: centered; `●` (#6b7280) when attachment present, empty when absent; `aria-label="Has attachment"` on the cell when present
- Cell padding: 8px 16px (sm/md tokens)
- Table width: 100%
- Column widths: Title auto (flex-grow), Created 140px, Updated 140px, File 60px

### Form Fields

- Same `.field` pattern as Phase 1 (label above input, 16px gap between fields)
- `<textarea>` for note body: min-height 240px, resize vertical, font-family inherited, font-size 16px, padding 8px 16px, border 1px solid #d1d5db, border-radius 6px, width 100% box-sizing border-box
- `<input type="file">` for attachment: displayed below a label "Attachment (optional)"; no custom styling — use browser native file input; font-size 14px
- Validation error messages: 14px color #dc2626, margin-top 4px, displayed below the relevant field

### Buttons and Links

- Primary button (submit, "New note"): background #2563eb, color #ffffff, border none, border-radius 6px, height 36px, padding 0 16px, font-size 14px weight 400, cursor pointer; hover background #1d4ed8; active background #1e40af
- Destructive action ("Delete"): plain text link or button styled as text; color #dc2626, font-size 14px, no background, no border; hover: text-decoration underline
- Muted link ("Cancel", "← Notes", "Sign out"): color #6b7280, font-size 14px, no underline; hover: color #111827
- Accent link ("Edit", attachment download, table title): color #2563eb, no underline; hover: underline

### Existing Attachment Display (Edit Form)

When editing a note that already has an attachment:
- Show text: "Current attachment: {filename}" — 14px color #6b7280
- Below it: a "Remove attachment" checkbox with label "Remove current attachment"
- Below it: file input labeled "Replace with new file (optional)"
- These three elements are grouped inside a `<fieldset>` or a `<div class="field">` with a descriptive label

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click note title in table | Navigate to note show page (`/notes/:id`) via Turbo Drive |
| Click "New note" button | Navigate to `/notes/new` via Turbo Drive |
| Submit "Create note" — valid | POST `/notes`; redirect to note show page; flash notice rendered on show page |
| Submit "Create note" — no body AND no attachment | Re-render form with validation error "A note must have a body or an attachment." below the form |
| Submit "Update note" — valid | PATCH `/notes/:id`; redirect to note show page |
| Submit "Update note" — invalid | Re-render edit form with validation errors |
| Click "Delete note" | `data-turbo-method="delete"` with `data-turbo-confirm` dialog; on confirm: DELETE `/notes/:id`; redirect to index |
| Click "← Notes" back link | Navigate to `/notes` via Turbo Drive |
| Click "Cancel" in form | Navigate to previous context (`/notes` from new form, `/notes/:id` from edit form) |
| Click "Download {filename}" | Browser download via Active Storage blob URL (`rails_blob_url`); no custom behavior |
| Check "Remove current attachment" | No JS needed; checkbox value submitted with form; Active Storage `_purge` param handles removal server-side |
| Access `/notes/:id` for another user's note | Controller scopes via `current_user.notes`; `ActiveRecord::RecordNotFound` raised → 404 page (D-10) |
| Unauthenticated access to any notes route | Redirect to sign-in page (inherited from `AuthenticatedController`, SEC-02) |
| Empty notes list | Show empty state (heading + body + "New note" link); no table rendered |
| Note with body only | Show page renders markdown; no attachment row rendered |
| Note with attachment only | Show page renders attachment row; no markdown body section rendered |
| Note with both body and attachment | Show page renders both sections |

**Turbo compatibility:** All standard Rails CRUD actions work with Turbo Drive. No `data-turbo="false"` needed. Delete uses `data-turbo-method="delete"` and `data-turbo-confirm`.

## Accessibility Contract

- All form inputs have associated `<label>` elements (Phase 1 pattern extended)
- Table has `<thead>` with `<th scope="col">` for each column
- Attachment indicator cell has `aria-label="Has attachment"` when `●` is rendered, `aria-label="No attachment"` when empty
- "New note" button is a standard `<a>` styled as button — keyboard focusable
- Delete confirmation uses browser native confirm dialog — keyboard accessible
- Color is not the sole indicator for any state (e.g., attachment indicator uses ● character, not only color)
- Focus ring on all interactive elements: `outline: 2px solid #2563eb; outline-offset: 2px` (inherited from Phase 1 input focus rule; extend to buttons and links)

## Implementation Notes for Executor

- All new CSS goes in `app/assets/stylesheets/application.css` (continuing Phase 1 pattern). Add clearly labeled sections with comments: `/* Notes layout */`, `/* Notes table */`, `/* Notes form */`, `/* Note show */`.
- Navigation bar: add HTML to `app/views/layouts/application.html.erb` inside `<body>`, wrapping `<%= yield %>` in a content div. Use `<% if user_signed_in? %>` to show nav only for authenticated users — sign-in page must remain unaffected.
- Date formatting: use Rails `l(note.created_at, format: :short)` or `note.created_at.strftime("%b %-d, %Y")`. Define a `:short` date format in `config/initializers/date_formats.rb` for reuse.
- Markdown body output: rendered HTML goes inside `<div class="note-body">` with `class` styles applied to standard HTML tags inside it. Use `raw sanitize(render_markdown(note.body), ...)` pattern from D-02.
- Page titles: set via `content_for(:title)` in each view, consumed by `<title>` in application layout.
- Delete button: use `button_to note_path(note), method: :delete, data: { turbo_confirm: "Delete this note? This cannot be undone." }, class: "btn-destructive"` — avoids link-with-GET-semantics for a destructive action.
- No JavaScript file changes needed for Phase 2. No Stimulus controller required.
- Active Storage: `rails active_storage:install && rails db:migrate` must run before notes routes are accessible. Verify `app/javascript/activestorage` is not needed (not using Direct Upload in Phase 2 — standard form upload only).
