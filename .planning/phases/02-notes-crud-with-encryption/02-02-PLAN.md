<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Rails | All form input (body text, file uploads, file_purge flag) is untrusted; crosses into NotesController via params. |
| Request -> Note row | `current_user.notes` scoping is the only thing preventing user A from creating/viewing user B's notes via the URL. |
| File upload -> Active Storage blob | Files of any type/size accepted in Phase 2 (per REQUIREMENTS NOTE-02). Stored on local disk. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Spoofing | Unauthenticated requests | mitigate | `NotesController < AuthenticatedController` inherits `before_action :authenticate_user!`; Devise enforces session. System test "unauthenticated visit redirects" proves it. |
| T-02-02-02 | Tampering | Mass-assignment of `user_id` via params | mitigate | `note_params` permits only `:body, :file, :file_purge` — `:user_id` is NOT permitted; user is set via `current_user.notes.build`. |
| T-02-02-03 | Information Disclosure | Cross-user note access via direct URL | mitigate | All queries scoped through `current_user.notes`. `find` on the scope raises `ActiveRecord::RecordNotFound` -> 404, never confirming existence (D-10). System test "user A does not see user B's notes" proves the index path; show path covered in Plan 03 threat model. |
| T-02-02-04 | Tampering | Markdown XSS via body | mitigate | `render_markdown` returns HTML, but Plan 03 will wrap it with `sanitize(..., tags: ..., attributes: ...)` per D-02 before output. This plan's index view does NOT render the body — only `derived_title` (plain text from the model) — so this slice is safe. Flag for Plan 03. |
| T-02-02-05 | Denial of Service | Unbounded file upload size | accept | No size limit in Phase 2 (deferred to Phase 3 polish). Local-disk development storage; production deployment is single-tenant per PROJECT.md. |
| T-02-02-06 | Tampering | CSRF on POST /notes | mitigate | Rails default `protect_from_forgery` in ApplicationController (Rails 8 default `ActionController::Base`); no change needed. System test uses Capybara which sends the CSRF token. |
| T-02-02-07 | Information Disclosure | Index query leaking other users' notes via JOIN bug | mitigate | `current_user.notes.with_attached_file.order(...)` — the `with_attached_file` eager-load uses Active Storage's join on `record_type='Note'` AND `record_id IN (...)` scoped via the user association, so other users' attachments cannot leak. |
| T-02-02-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this plan; redcarpet already added in Plan 01. Slopcheck not applicable. |
</threat_model>
