# Phase 1 — UI Design Contract

> Visual and interaction contract for the sign-in page only. Phase 1 delivers one UI surface: the Devise sign-in page. Generated by gsd-ui-researcher.

---

## Scope

This contract covers exactly one page: the Devise sign-in page (`/users/sign_in`).

No other views are in scope for Phase 1. All other UI surfaces (notes list, note detail, note form) belong to Phase 2. Polish and visual refinement belong to Phase 3.

---

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | No component library; plain CSS via Propshaft |
| Preset | not applicable | No shadcn; not a React/Next.js/Vite project |

**No "Forgot your password?" link in Phase 1.** D-04 defers ActionMailer setup; exposing the recovery link without a working mailer creates a broken UX. Webmaster uses Rails console for password resets in Phase 1. If Devise generates this link by default in the ERB template, it must be removed during customization (D-08).

**No "Sign up" or "Create account" link.** Self-registration is permanently out of scope (REQUIREMENTS.md Out of Scope). Any such link generated by Devise views must be removed during customization (D-08).

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Form submit — valid credentials | POST to Devise sessions controller; Turbo Drive handles response; redirect to notes list (root path) per D-07 |
| Form submit — invalid credentials | Devise re-renders sign-in page with `flash[:alert]` containing "Invalid email or password." (D-06); error appears below the form fields |
| Form submit — empty fields | Browser native `required` validation fires before POST; no custom client-side validation needed |
| Session active + visit sign-in | Devise redirects to root path (default Devise behavior) |
| Unauthenticated visit to protected route | Devise `authenticate_user!` redirects to sign-in page (SEC-02) |
| Browser close | Session ends; no persistent cookie (D-09, D-11) |

**Turbo compatibility:** The Devise sign-in form works with Turbo Drive out of the box. No `data-turbo="false"` attribute needed. Flash messages render correctly on the re-rendered page.
